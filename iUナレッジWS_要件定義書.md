## 1. これは何（概要）

### 背景
インターンで3人チームとして活動している。各メンバーが新しく学んだ専門用語を、3人みんなが見られる形で共有・保存できる場所が今はない。

### 目的
わからない専門用語を入力すると、AI が自動でリサーチして、わかりやすくイラストを多く使い、比喩表現（たとえ話）を使わずにまとめてくれる。さらに、似ている関連ワードも自動で提案してくれる。チーム3人だけの「私たちの辞書」を作る。

### 使う人（想定ユーザー）

## 2. スコープ

### やること
- 専門用語を入力して登録する。
- AI が用語の意味を自動でリサーチし、わかりやすいテキストとイメージイラストでまとめる（比喩表現なし）。
- 関連ワードを AI が自動で提案する。
- 登録した用語を3人で共有・閲覧・保存する。

- スマホ専用アプリ（iOS/Android ネイティブアプリ）は作らない。PC のブラウザ利用を前提とする（提案）。
- 用語以外の汎用チャット機能（雑談 AI）は作らない（提案）。
- 課金・決済機能は作らない（提案）。

## 4. 受け入れ基準（Given / When / Then）

### F-01 / F-02 用語の登録と AI リサーチ
- 正常系
  - Given：メンバーがログインして辞書画面を開いている
  - When：用語名（例「機械学習」）を入力して「登録」を押す
  - Then：AI が比喩表現なしの説明を生成し、用語が辞書に保存され、一覧に表示される
- 異常系（空入力）
  - Given：辞書画面を開いている
  - When：用語名を空のまま「登録」を押す
  - Then：「用語名を入力してください」とエラーが表示され、登録されない
- エッジケース（最大長）
  - Given：辞書画面を開いている
  - When：用語名に非常に長い文字列（例：上限を超える長さ）を入力する
  - Then：上限文字数を超える旨のメッセージが表示され、登録されない（上限値は TBD）

### F-03 イメージイラスト生成
- 正常系
  - Given：用語が登録された
  - When：AI がイラスト生成を実行する
  - Then：用語の説明と一緒にイラストが表示される
- 異常系（生成失敗）
  - Given：AI のイラスト生成サービスが応答しない
  - When：イラスト生成を試みる
  - Then：テキスト説明だけは表示し、イラスト部分は「画像を生成できませんでした（後でやり直せます）」と表示する

### F-04 関連ワードの自動提案
- 正常系
  - Given：用語の説明が生成された
  - When：画面に説明が表示される
  - Then：関連ワードが1件以上、一覧で表示される
- エッジケース（関連ワードなし）
  - Given：とても珍しい用語を登録した
  - When：関連ワードを探す
  - Then：「関連ワードは見つかりませんでした」と表示し、エラーにはしない

### F-05 用語一覧の閲覧（共有）
- 正常系
  - Given：メンバーAが用語を登録した
  - When：メンバーBが辞書画面を開く
  - Then：メンバーAが登録した用語がメンバーBの画面にも表示される
- 異常系（権限なし）
  - Given：チーム3人以外のユーザーがアクセスしようとする
  - When：辞書画面を開こうとする
  - Then：閲覧が拒否される（ログイン要求／アクセス拒否を表示）

### F-06 用語の検索
- 正常系
  - Given：複数の用語が登録されている
  - When：検索欄にキーワードを入力する
  - Then：キーワードに一致する用語だけが表示される
- エッジケース（該当なし）
  - Given：どの用語にも一致しないキーワード
  - When：検索する
  - Then：「該当する用語がありません」と表示する

## 6. 設計の概要

### アーキテクチャ
- ブラウザ（画面）：用語の入力・一覧・検索を表示する。
- サーバー（Next.js の API 部分）：登録リクエストを受け、AI API を呼び出して説明・イラスト・関連ワードを生成し、データベースに保存する。
- データベース（Supabase）：用語データを保存し、3人で共有する。
- 認証（Supabase Auth）：3人だけがログインして使えるようにする。

```
[ブラウザ] ⇄ [Next.js サーバー/API] ⇄ [AIサービス（テキスト/画像）]
                       ⇅
              [Supabase DB + 認証]
```

### 主要データモデル（提案・要確認）

term（用語）テーブル
| 項目名 | 型 | 説明 |
|---|---|---|
| id | UUID | 用語の一意なID |
| word | text | 用語名（必須） |
| description | text | AI が生成した比喩なしの説明 |
| image_url | text | 生成イラストの保存先URL（無い場合あり） |
| created_by | UUID | 登録した人（user の id） |
| created_at | timestamp | 登録日時 |
| updated_at | timestamp | 更新日時 |

related_word（関連ワード）テーブル
| 項目名 | 型 | 説明 |
|---|---|---|
| id | UUID | 一意なID |
| term_id | UUID | どの用語に対する関連ワードか（term.id） |
| related_text | text | 関連ワードの語 |

user（利用者）※Supabase Auth が管理
| 項目名 | 型 | 説明 |
|---|---|---|
| id | UUID | ユーザーID |
| email | text | ログイン用メール |
| name | text | 表示名 |

関係：user 1 ─ 多 term（1人が複数の用語を登録）／ term 1 ─ 多 related_word（1用語に複数の関連ワード）。

### 画面・ファイル構成（例・提案）
```
iU-knowledge-ws/
├─ app/
│  ├─ page.tsx              # 用語一覧（トップ画面）
│  ├─ terms/
│  │  ├─ new/page.tsx       # 用語の新規登録画面
│  │  └─ [id]/page.tsx      # 用語の詳細（説明・イラスト・関連ワード）
│  ├─ login/page.tsx        # ログイン画面
│  └─ api/
│     ├─ terms/route.ts     # 用語の登録・取得API
│     └─ ai/route.ts        # AIリサーチ・イラスト・関連ワード生成API
├─ lib/
│  ├─ supabase.ts           # DB接続設定

├─ components/              # 画面の部品（一覧カードなど）

└─ package.json
```

### 2026-06-13 イラストを「絵で説明する図解」に改善＋画質を medium に

- **きっかけ**: 生成されるイラストが「分かりづらい」。イラストの役割は“その用語を絵で分かりやすく説明する”こと。
- **やったこと（プロンプト）**: ただの雰囲気イラストではなく、**仕組み・構造・流れを図解で示す**指示に作り替えた（`lib/ai.ts`）。
  - まず安いテキストAI(nano)に「概念を図解で説明する絵の設計（英語）」を書かせ、その設計を画像AIに渡す“二段構え”にした。
  - 画像AIは文字を崩して描く（特に日本語）ので、**文字は入れず、矢印・アイコン・箱（入力→処理→出力 など）で意味を伝える**よう指定。
- **やったこと（画質）**: 直前に料金優先で `low` にしていたが、図解は解像度が要るため **`medium` を既定**に変更（1枚 約8円。`low`=約0.9円／`high`=約32円。`OPENAI_IMAGE_QUALITY` で変更可）。
- **確認したこと**: 型チェック エラー0。実際の見え方は開発サーバーで要確認（料金が出るのは画像生成時）。

### 2026-06-13 AIモデルを「料金最優先」で見直し

- **やったこと**: 最新の料金・性能を調べ（`docs/research` ではなくこの場で比較）、使うモデルを安い構成に切り替えた。
  - テキスト（説明・タグ・議事録抽出）: `gpt-5.4-mini` → **`gpt-5.4-nano`**（約7割安。短い日本語＋JSONには十分）。
  - 画像（イラスト）: `gpt-image-2` のまま、**品質を `low` に固定**。ここが料金の一番動く所（low ≒ 0.9円／medium ≒ 8円／high ≒ 32円 per枚）。きれいにしたい時は `OPENAI_IMAGE_QUALITY=medium` にすればよい。
  - 将来の議事録文字起こし: 予定を `gpt-realtime-whisper` → **`gpt-4o-mini-transcribe`**（約半額・$0.003/分）に変更。
- **決めたこと・理由**: 3人・少量利用ではテキスト代は誤差（月数十円）。コストが実際に効くのは画像なので、画像品質を下げるのが一番効く。テキストは乗り換えの手間を避けてOpenAIのまま nano に。
- **どこを直したか**: 既定値は `lib/config.ts`（`TEXT_MODEL` / `IMAGE_QUALITY` / `TRANSCRIBE_MODEL`）。`.env.local` で上書き可能（`OPENAI_TEXT_MODEL` / `OPENAI_IMAGE_QUALITY` / `OPENAI_TRANSCRIBE_MODEL`）。
- **注意**: もし `.env.local` に `OPENAI_TEXT_MODEL=gpt-5.4-mini` 等が書いてあると、そちらが優先される。安くするなら該当行を消すか nano に直す。
- **確認したこと**: 型チェック エラー0。

### 2026-06-13 見た目をプロ級に刷新＋機能を4つ追加

- **やったこと（見た目）**:
  - アプリ全体の見た目を「クリーン・ミニマル」（Notion / Linear のような、余白を活かした落ち着いた雰囲気）に作り替えた。色・余白・角丸などの“決まりごと”を1か所（`app/globals.css`）にまとめ、各画面はその決まりだけを使うようにした。配色を変えたくなっても、基本この1ファイルを直すだけで全体に反映される。
  - よく使う部品（ボタン・カード・入力欄・タグ・読み込み中の仮表示など）を `components/ui/` に共通化した。
- **やったこと（新機能4つ）**:
  1. **ダークモード**: 画面右上の月/太陽ボタンで切替。選んだ状態は記憶され、次に開いたときも最初から正しい明るさで出る（パッと切り替わる際のチラつきなし）。
  2. **タグ分類＋絞り込み**: 用語に「分野タグ」（例：AI / 統計 / Web）が付く。AIが登録時に自動で付け、編集でも直せる。一覧の上のタグを押すと、その分野だけに絞り込める。
  3. **関連ワードの相互リンク**: 関連ワードがすでに辞書にある用語なら、その用語ページへ直接ジャンプできる。さらに詳細ページに「この用語とつながる用語」（自分を関連ワードに挙げている用語）も表示し、辞書が“つながり”として育つようにした。
  4. **議事録から用語を自動登録**: ヘッダーの「議事録から登録」から、会議メモなどを貼り付けると、AIが専門用語を見つけて、まとめて辞書に登録する（1回で最大12件）。
- **使い心地の工夫**:
  - 用語を登録してAIが考えている数十秒の間、完成後の見た目を“うすい仮表示（スケルトン）”で先に見せ、いま何をしているか（意味を調べる→関連ワード→説明→イラスト）を順に表示。待ち時間が短く感じられ、安心できる。
  - からっぽの画面（用語ゼロ・検索0件）も、次にやることが分かる案内に作り込んだ。
- **決めたこと・理由**:
  - 見た目の方向は「クリーン・ミニマル」、アクセント色はインディゴ（青紫）に決定（リサーチして比較した結果。詳細は `docs/research/2026-06-13_UI-UXモダン化リサーチ.md`）。
  - 議事録の一括登録は、料金を抑えるため1回の件数に上限を設け、重いイラスト生成はその場では行わない（各用語のページから後で作れる）。録音→文字起こしは今回は入れず、まずは文章貼り付けから。
- **確認したこと**: 型チェック（エラー0）・テスト（16件パス）・本番ビルド成功。開発サーバーを起動し、未ログインは `/login` へ自動で飛び、ログイン後はトップ・新規登録・議事録ページが正しく表示され、ダークモードの切替も効くことを確認した。
- **暫定対応（前回から継続）**: データ保存はローカルファイル、ログインは簡易版のまま（DB・本認証が決まったら差し替える）。
- **次にやること**: データベース（保存先）と本認証を決める。タグの自動付与や議事録抽出の精度を実データで確かめる。

### 2026-06-12 アプリの土台づくり（Next.js 16 + OpenAI）

- **やったこと**:
  - 「私たちの辞書」アプリの土台を作った。用語を登録すると、AIが①比喩を使わない説明文②関連ワード③イメージイラストを自動で作って保存する。一覧・検索・編集・削除もできる。
  - まだログイン・登録・一覧・検索・編集・削除の画面が一通り動く状態（AIはOpenAIキーを入れれば動く）。
- **決めたこと・理由**:
  - 技術は TypeScript + Next.js + OpenAI + Vercel に決定。データベース（保存先）は**まだ保留**。
  - 保留中でもアプリを動かせるよう、データは当面ローカルのファイル（`.data/terms.json`）にためる。あとでDBが決まったら、保存まわり（`lib/store/`）だけ差し替えれば画面は変えずに済むようにした。
  - AIテキストは安い `gpt-5.4-mini` を初期値に（モデル名は `.env.local` の1行で変更可）。画像は `gpt-image-2`。
  - 将来の「議事録から専門用語を自動抽出」に備え、`lib/ai.ts` に抽出関数の土台 `extractTermsFromText` を先に入れておいた（議事録の文字起こしは OpenAI の `gpt-realtime-whisper` を使う予定）。
- **確認したこと**: 型チェック（エラー0）・テスト（4件パス）・本番ビルド成功。開発サーバーを起動し、未ログインは `/login` へ自動で飛び、ログインすると空の用語一覧が表示されることを確認（要件定義書フェーズ1のゴール達成）。
- **暫定対応**:
  - データ保存がローカルファイル → DB（Supabase 等）が決まったら差し替える。
  - ログインは「名前＋合言葉」をクッキーに入れるだけの簡易版（`lib/auth.ts`）→ 本認証（Supabase Auth 等）が決まったら差し替える。
- **次にやること**:
  - データベース（保存先）を決める。
  - 用語名・説明文の最大文字数など、要件定義書のTBDを確定する。
  - 議事録機能の画面を追加する。

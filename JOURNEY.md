### 2026-07-02 用語詳細で「＜ ＞」の前後めくりを追加（並び順・絞り込みを引き継ぎ）＋本番公開

- **やったこと**:

  2. **ホームで選んだ並び順（名前順・いいね順など）や検索・タグ絞り込みを、そのまま詳細ページへ引き継ぐ**ようにした。一覧で見ていたのと同じ順番・同じ絞り込みの中でめくれる。めくっても順番は保たれ、「一覧へ戻る」も同じ条件の一覧に戻る。
- **どこを直したか**: 新規 `components/TermPager.tsx`（＜ ＞ ボタン＋左右キー）／`app/terms/[id]/page.tsx`（一覧と同じ手順で並びを組み直し前後を計算・文脈を引き継ぎ）／`app/page.tsx`・`components/TermCard.tsx`・`components/TaikoSelector.tsx`（用語リンクに並び順・絞り込みの文脈を付与）。

### 2026-07-02 「用語が削除できない」不具合を修正（埋め込み枠でも消せるように）

- **きっかけ**: 登録した用語の「削除」ボタンを押しても消えない、という報告。
- **原因**: 削除ボタンは、押したときにブラウザ標準の確認ダイアログ（`window.confirm`＝「本当に削除しますか？ OK / キャンセル」の小窓）を出し、OKのときだけ送信する作りだった。ところがこのアプリはプラットフォームの**別ドメインの枠（iframe）**の中で開かれる。この枠の中では、ブラウザが安全のために確認ダイアログを**無視して常に「キャンセル」扱い**にする。そのため確認の小窓が出ないまま送信もキャンセルされ、いつまでも削除できなかった。※ログインをクッキーからやめたのと同じ「枠の中では使えない機能があった」問題。
- **やったこと**: ブラウザ標準の確認ダイアログをやめ、**画面の中の2段階ボタン**で確認するようにした。最初は「削除」ボタンだけを出し、押すと「『◯◯』を削除しますか？元に戻せません。」という文と「削除する／やめる」ボタンに変わる。「削除する」を押したときだけ実際に消える（消している間は「削除中…」表示で二重押し防止）。枠の中でも確実に動く作り。
- **どこを直したか**: `components/DeleteTermButton.tsx`（用語の削除）。同じ作りで動かなかった `components/DeleteMinuteButton.tsx`（議事録の削除）も同時に直した。
- **確認したこと**: 型チェック エラー0／テスト72件パス。
- **メモ**: 削除の中身（サーバー側の処理・データの消去）はもともと正しく、原因はボタンの「確認のさせ方」だけだった。同じ理由で「確認ダイアログ」を使っている箇所が他に出たら、同じ2段階ボタン方式に置き換える。

- **暫定対応**: 入口の認証は無し（内輪利用が前提）。名前はブラウザごとの記憶＝端末を変えると入れ直し。将来プラットフォーム共通ログインを作る場合はそれに合わせて記名を差し替え。

### 2026-06-21 用語のイラストをクラウドの画像置き場（Cloud Storage）へ

- **きっかけ**: クラウドに上げたナレッジWSは、用語のイラストが表示されなかった。原因は、AIが作ったイラストを**そのサーバーの中**に置く作りだったこと。クラウド（Cloud Run）はアクセスが無いと電源が切れ、再び動くときは**中身がまっさら**に戻るため、せっかく作った画像が消えてしまう。
- **やったこと**: イラストの置き場を、消えない**クラウドの画像置き場（Google Cloud Storage のバケット `iu-internship-assets`）**に変えた。
  1. 画像を「保存する・読み出す・消す」係（`lib/images.ts`）を、設定 `ASSETS_BUCKET` があれば**クラウドの置き場**へ、無ければ**今までどおり自分のPCの中**へ、と自動で切り替わるようにした（持ち帰ってローカルで動かすときはそのまま動く）。
  2. 画像を画面に届ける係（`app/illustrations/[id]/route.ts`）も、置き場が変わっても画面側は手直し不要なように整理した。
  3. これまでに作りためた**既存のイラスト56枚**を、一度だけクラウドの置き場へアップロードした。
- **どこを直したか**: `lib/images.ts`（クラウド/ローカルの切り替え・読み出し関数を追加）／`app/illustrations/[id]/route.ts`（その読み出し関数を使う形に統一）／`package.json`・`package-lock.json`（クラウド置き場の部品 `@google-cloud/storage` を追加）。
- **確認したこと**: 型チェック エラー0／本番ビルド成功。クラウド版（`iu-knowledge-ws`）を新しいコードで再公開し、設定 `ASSETS_BUCKET=iu-internship-assets` を追加。公開URLでイラスト2枚に実際にアクセスし、**どちらも画像（PNG・約860KB／940KB）が返ってくる**ことを確認＝クラウドでも用語の絵が出るようになった。
- **暫定対応**: 画像はアプリ経由で配信（置き場は非公開のまま）。今のところ容量はごくわずか（数十MB）で費用はほぼ0。

### 2026-06-18 用語一覧：グリッドの列数を変えられるように＋表示デザインの切り替え

- **きっかけ**: 「グリッド表示のとき、1行に並べる数を変えられるようにして」という要望から始まり、操作方法や見た目を相談しながら何度か作り直して今の形になった。
- **やったこと**:
  1. **列数を変えられる**ように。グリッド表示のとき、右上の「− ＋」ボタンで1行に並べる枚数を **2〜6** の間で増減できる（**−で減る・＋で増える**）。最小2・最大6まで来ると、そのボタンは押せない見た目（うすいグレー）になる。既定は3列（これまでと同じ見た目）。
  2. **表示デザインの切り替えボタン**をヘッダー右上に追加。ワンタッチで2つの見た目を行き来できる。
     - **クラシック**: 中身を画面中央に寄せた、読みやすい幅。
     - **ワイド**: 画面の横幅いっぱいに広げ、画面サイズに合わせて左右の余白が変わる（レスポンシブ）。
     - 列数の「− ＋」はどちらの見た目でも共通。
  3. 選んだ**デザインも列数も覚えておく**ので、次に開いても同じ状態で表示される（デザインは1年保存。テーマのダーク／ライト切り替えと同じ作り方なので、開いた瞬間のちらつきが出ない）。

### 2026-06-18 タグの絞り込みを「複数選択＋もう一度押すと解除」に

- **きっかけ**: 「有効化しているタグをもう一度クリックして、他に選んでいるタグが無ければ『すべて』に戻る挙動にして」という要望。これまではタグは1つだけの絞り込みで、選んでいるタグを押しても何も起きなかった。
- **やったこと**: タグの絞り込みを**複数選べる**ようにし、チップを**トグル（押すたびにON/OFF）**にした。
  - 押すと、そのタグが選択に**入っていなければ追加・入っていれば解除**。
  - 解除した結果、**選んでいるタグが1つも無くなったら自動で「すべて」に戻る**。他に選んでいるタグが残っていれば、それで絞り込んだままにする。
  - 複数選んだときは「**どれか1つでも当てはまる**用語」を表示（和集合）。用語は1つにつき1タグなので、「全部持つ」だと2つ選ぶと0件になってしまうため、こちらの方が自然。
- **URLの持ち方**: これまでの `?tag=AI`（1つ）から `?tags=AI,Web`（カンマ区切りで複数）に変更。古い `?tag=` のリンクも読めるよう後方互換を残した。検索・並び順・表示との併用もこれまで通り保たれる。
- **どこを直したか**: `lib/terms.ts`（`filterByTags`＝和集合・`toggleTag`＝ON/OFF・`parseTagsParam`＝URL読み取り を追加）／`components/TagFilter.tsx`（トグル＆複数対応）／`app/page.tsx`（複数タグで絞り込み）／`components/SearchBar.tsx`・`components/TermsToolbar.tsx`（条件の受け渡しを tags に）／`app/terms/[id]/page.tsx`（タグのリンクを `?tags=` に）。テストは `tests/tag-multi.test.ts` を追加し、`tests/url.test.ts` を更新。
- **確認したこと**: 型チェック エラー0／テスト72件パス／本番ビルド成功。実機（ポート8530）で、`?tags=AI` のときAIチップを押すとリンク先が `/`（＝すべてに戻る）、`?tags=AI,Web` のときAIを押すと `?tags=Web`（AIだけ外れて残りは維持）になることを、実際のリンクで確認。

### 2026-06-18 タグをAIで自動化（既存も一括付与）＋太鼓UIは「クリックで閉じる」

- **きっかけ**: 「いまある用語すべてにタグを付けたい」「新しい用語はAIが既存の選択肢の中から自動でタグを選んでほしい」「太鼓UIで開いた札をもう一度クリックしたら閉じてほしい」という3つの要望。

### 2026-06-13 見た目をプロ級に刷新＋機能を4つ追加

- **やったこと（見た目）**:
  - アプリ全体の見た目を「クリーン・ミニマル」（Notion / Linear のような、余白を活かした落ち着いた雰囲気）に作り替えた。色・余白・角丸などの“決まりごと”を1か所（`app/globals.css`）にまとめ、各画面はその決まりだけを使うようにした。配色を変えたくなっても、基本この1ファイルを直すだけで全体に反映される。
  - よく使う部品（ボタン・カード・入力欄・タグ・読み込み中の仮表示など）を `components/ui/` に共通化した。
- **やったこと（新機能4つ）**:
  1. **ダークモード**: 画面右上の月/太陽ボタンで切替。選んだ状態は記憶され、次に開いたときも最初から正しい明るさで出る（パッと切り替わる際のチラつきなし）。
  2. **タグ分類＋絞り込み**: 用語に「分野タグ」（例：AI / 統計 / Web）が付く。AIが登録時に自動で付け、編集でも直せる。一覧の上のタグを押すと、その分野だけに絞り込める。
  3. **関連ワードの相互リンク**: 関連ワードがすでに辞書にある用語なら、その用語ページへ直接ジャンプできる。さらに詳細ページに「この用語とつながる用語」（自分を関連ワードに挙げている用語）も表示し、辞書が“つながり”として育つようにした。
  4. **議事録から用語を自動登録**: ヘッダーの「議事録から登録」から、会議メモなどを貼り付けると、AIが専門用語を見つけて、まとめて辞書に登録する（1回で最大12件）。
- **使い心地の工夫**:
  - 用語を登録してAIが考えている数十秒の間、完成後の見た目を“うすい仮表示（スケルトン）”で先に見せ、いま何をしているか（意味を調べる→関連ワード→説明→イラスト）を順に表示。待ち時間が短く感じられ、安心できる。
  - からっぽの画面（用語ゼロ・検索0件）も、次にやることが分かる案内に作り込んだ。
- **決めたこと・理由**:
  - 見た目の方向は「クリーン・ミニマル」、アクセント色はインディゴ（青紫）に決定（リサーチして比較した結果。詳細は `docs/research/2026-06-13_UI-UXモダン化リサーチ.md`）。
  - 議事録の一括登録は、料金を抑えるため1回の件数に上限を設け、重いイラスト生成はその場では行わない（各用語のページから後で作れる）。録音→文字起こしは今回は入れず、まずは文章貼り付けから。
- **確認したこと**: 型チェック（エラー0）・テスト（16件パス）・本番ビルド成功。開発サーバーを起動し、未ログインは `/login` へ自動で飛び、ログイン後はトップ・新規登録・議事録ページが正しく表示され、ダークモードの切替も効くことを確認した。
- **暫定対応（前回から継続）**: データ保存はローカルファイル、ログインは簡易版のまま（DB・本認証が決まったら差し替える）。
- **次にやること**: データベース（保存先）と本認証を決める。タグの自動付与や議事録抽出の精度を実データで確かめる。

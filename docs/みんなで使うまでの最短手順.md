# みんなで使えるソフトにするための最短手順

> このメモは、エンジニアでない人にも分かるように書いています。専門用語はできるだけ避け、たとえ話は使いません。
> 対象：iUナレッジWS を「自分のPCで動く試作品」から「チーム・社内の数人が同じものを使えるソフト」にしたい人。
> 作成日：2026-06-15

---

## 結論（最短ルートは3つの作業だけ）

いまのアプリは **作った人のPCの中だけ** で動いています。みんなで使えるようにするには、次の3つを変えれば十分です。

1. **データの保存先を、PCの中から「クラウド上のデータベース」に移す**
2. **ログインを、仮のものから「本物のログイン」に変える**
3. **アプリを「公開する場所（クラウド）」に置いて、URLで開けるようにする**

おすすめの組み合わせは **Supabase（保存とログインを担当）＋ Vercel（公開する場所）** です。
このアプリは最初から、この組み合わせに差し替えやすい作りにしてあります。

---

## いまの状態（正直なところ）

| 項目 | いまどうなっているか | みんなで使うときの問題 |
|---|---|---|
| 動く場所 | 作った人のPCの中だけ | 他の人は開けない。PCを閉じると止まる |
| データの保存先 | PC内のファイル（`.data/terms.json`） | 他の人とデータを共有できない |
| イラストの保存先 | PC内のフォルダ（`public/generated/`） | 同上。公開クラウドでは保存が消えてしまう |
| ログイン | 「名前＋合言葉」を確認するだけの仮のもの | 誰でも合言葉を知っていれば入れる。社内利用には弱い |

> つまり「機能はだいたい完成しているが、保存とログインと置き場所が“仮”のまま」という状態です。
> 機能の作り直しは不要で、この3つの“仮”を本物に替えるのが最短です。

---

## なぜこの3つが必要なのか（簡単な理由）

- **保存先**：いまはデータが各PCの中にあるので、3人で同じ辞書を見られません。みんなが同じデータを見るには、1か所（クラウド上のデータベース）に置く必要があります。
- **ログイン**：いまの仮ログインは合言葉だけなので、社内で安心して使うには「誰がメンバーか」をきちんと管理できる本物のログインが要ります。
- **公開する場所**：アプリを誰かのPCではなくクラウドに置くと、URLを開くだけで全員が同じものを使えます。ただしクラウドでは「PC内に保存する」やり方が使えないため、上の保存先の移行とセットで行います。

---

## 最短ルートの手順（おすすめ：Supabase ＋ Vercel）

順番に進めれば、途中でつまずきにくくなります。各ステップに「誰が決める／作業するか」を添えました。

### ステップ1：保存先とログインのサービスを用意する
- **Supabase**（無料枠あり）のアカウントを作り、プロジェクトを1つ作ります。
- データの置き場所（リージョン）は **東京** を選ぶと、社内利用で扱いやすいです。
- Supabase は「データベース」と「ログイン」と「画像の保存」を1つでまかなえます。
- 👉 決めること：このサービスを使ってよいか（→「あなたが決めること」の表へ）。

### ステップ2：アプリの“差し替え口”を本物につなぐ（エンジニア作業）
このアプリは、差し替える場所が最初から決まっています。エンジニア（またはAIに手伝ってもらって）が次を行います。
- データ保存：`lib/store/` に Supabase 版の保存処理を追加し、`lib/store/index.ts` で切り替える。**画面のコードは変えなくてOK**。
- ログイン：`lib/auth.ts` を Supabase のログインに差し替える。
- イラスト保存：`lib/images.ts` を Supabase の画像保存（ストレージ）に差し替える。

> ここが唯一のプログラム作業です。画面や機能はそのまま使えるので、作業範囲は限定的です。

### ステップ3：秘密の設定（鍵）を登録する
- OpenAI の API キーや Supabase の接続情報などの「鍵」を、Vercel の設定画面に登録します。
- 鍵はファイルに書かず、必ずこの設定画面（または `.env.local`）にだけ入れます。

### ステップ4：Vercel で公開する
- **Vercel**（公開する場所）にこのプロジェクトをつなぎ、公開します。
- 公開が終わると `https://〇〇.vercel.app` のような URL が発行され、URL を開けば誰でも同じアプリを使えるようになります。

### ステップ5：メンバーを招待して使い始める
- Supabase のログインに、使う人のメールアドレスを登録（招待）します。
- 招待された人だけがログインでき、全員が同じ辞書を見られます。

## 社内で使うために、追加で考えること

社内利用では「機能が動く」だけでなく、**誰が入れるか・データがどこにあるか・費用** をはっきりさせておくと安心です。

### 1. 誰が入れるか（アクセスの制限）
- いちばん簡単：Supabase のログインに **社員のメールアドレスだけを招待** する（招待制）。
- もっと安全にしたいなら：会社が **Google Workspace か Microsoft 365** を使っている場合、その会社アカウントでログインできるようにすると、「会社のアカウントを持つ人だけ」に自然に限定できます（新しい合言葉を配る必要がない）。
- さらに念のため：URL を知られても勝手に開けないよう、Vercel 側でも閲覧制限（パスワード保護など）をかけられます。

### 2. データの置き場所と扱い
- 用語データやイラストは Supabase（東京リージョンを選べば国内）に保存されます。どこに保存されるかを社内で共有しておきましょう。
- AI を使うとき、入力した用語名などは **OpenAI に送られます**。OpenAI の API は、送ったデータを学習に使わない方針ですが、念のため **社外秘の固有名詞や機密情報はそのまま入力しない** ルールにしておくと安全です。
- データの取り扱いをより厳しく管理したい場合は、OpenAI の代わりに **Azure OpenAI（Microsoft 経由）** を使う選択肢もあります（今回の最短ルートからは外れるので、必要になったら検討）。

### 3. 費用の管理
- AI の利用料は **使った分だけ** かかります。画面右上のコインのボタンで「1件あたりの目安」が見られます（おおよそ：用語1件の登録 ≒ 数円〜十数円、内訳の大半はイラスト）。
- OpenAI 側で **月の利用上限** を設定しておくと、想定外の請求を防げます。
- 公開する場所（Vercel）と保存先（Supabase）には無料枠がありますが、**会社の業務で使う場合は有料プランが必要になることがあります**（Vercel は商用利用だと有料プラン、目安 月20ドル程度）。少人数なら無料枠で足りる可能性もあるので、まず無料で始めて、足りなくなったら切り替えるのが現実的です。

### 4. アカウントの管理（運用面）
- 誰が使えるかの一覧（招待したメンバー）を、管理する人を1人決めておきます。
- メンバーが入れ替わったり退職したときに、**ログインを止める（招待を取り消す）** 手順を決めておきます。

## だいたいの手間と費用の感覚（目安）

正確な数字ではなく、規模感をつかむための目安です。

- **作業の手間**：機能は完成しているので、上のステップ2（差し替え）が中心。AI の手を借りれば、半日〜数日が目安。
- **公開作業（ステップ3〜5）**：慣れていれば1〜2時間ほど。
- **毎月の費用**：少人数なら無料枠で始められる可能性あり。きちんと業務利用するなら、Vercel・Supabase の有料プラン（各 月20〜25ドル程度）＋ AI の利用料（使った分だけ）。

---

## 今回はやらないこと（範囲外）

- スマホ専用アプリ化（PCのブラウザ利用が前提）
- 社外への一般公開・誰でも登録できる仕組み
- 会議の録音から自動で文字起こしする機能（いまは文章の貼り付けからの登録のみ）

## まとめ

- 機能はほぼ完成。残りは **保存先・ログイン・公開場所** の3つを“仮”から“本物”に替えるだけ。
- 最短は **Supabase ＋ Vercel**。差し替え口は用意済みなので、画面の作り直しは不要。
- 社内利用では **誰が入れるか・データの置き場所・費用** をあわせて決めておくと安心。
- まず上の「あなたが決めること」を埋めれば、すぐ作業に入れます。
